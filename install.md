# Membersly - Инструкция по установке и развертыванию

Данное руководство предназначено для системных администраторов и разработчиков, разворачивающих Membersly.

## 0. Автоматическая установка (One-Line)

Если у вас установлен Docker и Git, вы можете развернуть проект одной командой:

```bash
curl -sSL https://raw.githubusercontent.com/kornalexandr2/Membersly/master/install.sh | bash
```

---

## 1. Подготовка системы

### 1.1. Требования
- **Docker & Docker Compose:** Проект полностью контейнеризирован.
- **Доменное имя:** Необходимо для работы WebApp внутри Telegram и приема Webhook-уведомлений от платежных систем (рекомендуется использование SSL/HTTPS).

### 1.2. Регистрация Главного Бота
1. Создайте бота через [@BotFather](https://t.me/botfather).
2. Получите **API TOKEN**. Этот токен будет использоваться как "Мастер-ключ" для первичного запуска.
3. В меню `/setwebapp` укажите URL вашего будущего фронтенда (например, `https://membersly.yourdomain.com`).

---

## 2. Быстрый старт (Docker)

### 2.1. Клонирование
```bash
git clone https://github.com/kornalexandr2/Membersly.git
cd Membersly
```

### 2.2. Конфигурация (.env)
Создайте файл `backend/.env`. Если поля оставить пустыми, проект запустится на демонстрационных значениях, но функционал платежей и Telegram будет ограничен.

```env
# Обязательно для работы Telegram
BOT_TOKEN=ваш_токен

# Настройки БД (внутри Docker можно оставить как есть)
DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/membersly
REDIS_URL=redis://redis:6379/0

# Платежи (ЮKassa)
YOOKASSA_SHOP_ID=
YOOKASSA_SECRET_KEY=
```

### 2.3. Запуск
```bash
docker-compose up -d --build
```
После запуска все таблицы в базе данных будут созданы автоматически при старте сервиса `backend`.

---

## 3. Настройка через интерфейс

После запуска и применения миграций:
1. Откройте адрес вашего сервера в браузере.
2. Авторизуйтесь под стандартными учетными данными администратора:
   - **Логин:** `admin`
   - **Пароль:** `admin1234`
3. **Управление ботами:** В разделе "Настройки" вы можете добавить дополнительные токены ботов.
4. **Управление каналами:** Добавьте ID каналов или групп. Бот автоматически предложит назначить его администратором для управления доступом.
