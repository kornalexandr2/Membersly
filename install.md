# Membersly - Инструкция по установке и развертыванию

Данное руководство предназначено для системных администраторов и разработчиков, разворачивающих Membersly.

## 0. Автоматическая установка (One-Line)

Если у вас установлен Docker и Git, вы можете развернуть проект одной командой:

```bash
curl -sSL https://raw.githubusercontent.com/kornalexandr2/Membersly/master/install.sh | bash
```

---

## 1. Подготовка системы

### 1.1. Требования
- **Docker & Docker Compose:** Проект полностью контейнеризирован.
- **Доменное имя:** Необходимо для работы WebApp внутри Telegram и приема Webhook-уведомлений от платежных систем (рекомендуется использование SSL/HTTPS).

### 1.2. Регистрация Главного Бота
1. Создайте бота через [@BotFather](https://t.me/botfather).
2. Получите **API TOKEN**. Этот токен будет использоваться как "Мастер-ключ" для первичного запуска.
3. В меню `/setwebapp` укажите URL вашего будущего фронтенда (например, `https://membersly.yourdomain.com`).

---

## 2. Быстрый старт (Docker)

### 2.1. Клонирование
```bash
git clone https://github.com/kornalexandr2/Membersly.git
cd Membersly
```

### 2.2. Конфигурация (.env)
Создайте файл `backend/.env` (вы можете скопировать пример `cp backend/.env.example backend/.env` и отредактировать его командой `nano backend/.env`). 

Если поля оставить пустыми, проект запустится на демонстрационных значениях, но функционал платежей и Telegram будет ограничен.

```env
# Обязательно для работы Telegram
BOT_TOKEN=ваш_токен

# Настройки БД (внутри Docker можно оставить как есть)
DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/membersly
REDIS_URL=redis://redis:6379/0

# Платежи (ЮKassa)
YOOKASSA_SHOP_ID=
YOOKASSA_SECRET_KEY=
```

### 2.3. Запуск
```bash
docker-compose up -d --build
```
После запуска сервисы будут доступны по IP-адресу вашего сервера:
- **Frontend (Web UI):** `http://<ВАШ_IP>:6116`
- **Backend (API):** `http://<ВАШ_IP>:8000`

## 3. Настройка Reverse Proxy (Nginx Proxy Manager / Nginx)

Так как проект запускается по IP-адресам и портам, вам необходимо самостоятельно настроить проксирование трафика с вашего домена на нужные контейнеры, а также выпустить SSL сертификат.

### Пример для Nginx Proxy Manager (NPM):
1. Добавьте новый **Proxy Host**.
2. В поле **Domain Names** введите ваш домен (например, `membersly.yourdomain.com`).
3. В поле **Forward Hostname / IP** введите локальный IP адрес вашего сервера (не используйте `localhost`, используйте реальный IP в локальной сети, например `192.168.1.100` или IP интерфейса `docker0` - `172.17.0.1`).
4. В поле **Forward Port** укажите `6116` (порт фронтенда).
5. Перейдите во вкладку **Custom Locations**:
   - Нажмите **Add location**.
   - В поле **Location** введите: `/api/` (обязательно со слешами).
   - В поле **Forward Hostname / IP** введите тот же локальный IP сервера.
   - В поле **Forward Port** укажите `8000`.
   - Нажмите значок шестеренки (Advanced) рядом с Location и добавьте туда правило обрезки пути (чтобы бэкенд не получал префикс `/api/`), или просто укажите в качестве адреса назначения URL со слешем на конце (зависит от версии NPM). Оптимальный вариант конфигурации для Nginx: `proxy_pass http://<ВАШ_IP>:8000/;` (со слешем на конце).
6. Во вкладке **SSL** выпустите новый сертификат Let's Encrypt и сохраните.

## 4. Шаги после установки (Post-Installation)

Сразу после успешного запуска проекта выполните следующие шаги для корректной настройки системы:

### Шаг 1. Первичная авторизация
1. Откройте адрес вашего сервера в браузере (например, `http://ваш-домен/`).
2. В верхнем меню выберите **Admin** (Control).
3. Авторизуйтесь под стандартными учетными данными администратора:
   - **Логин:** `admin`
   - **Пароль:** `admin1234`

### Шаг 2. Безопасность
1. Перейдите во вкладку **Settings**.
2. Измените стандартный пароль (`admin1234`) на надежный.
3. Введите свой `Telegram ID` (можно узнать у бота `@userinfobot`), чтобы получать уведомления о новых продажах прямо в Telegram.

### Шаг 3. Подключение и настройка Бота
1. Перейдите во вкладку **Bots**.
2. Вставьте ваш токен от `@BotFather` и нажмите **Authorize**. *Бот начнет работать немедленно.*
3. Убедитесь, что в Telegram для вашего бота включен WebApp (`/setwebapp` в `@BotFather`) со ссылкой на ваш сервер.

### Шаг 4. Добавление ресурсов (Каналы/Группы)
1. Добавьте вашего бота в нужный Telegram-канал или группу в качестве **Администратора** с правами:
   - Добавление участников (пригласительные ссылки).
   - Удаление пользователей (бан).
   - Закрепление сообщений.
2. Перейдите во вкладку **Channels** в админке.
3. Добавьте `Chat ID` вашего канала (обычно начинается с `-100...`), название и настройте приветственное сообщение.

### Шаг 5. Создание тарифов
1. Перейдите во вкладку **Tariffs**.
2. Создайте тарифный план (укажите цену, длительность, наличие пробного периода).
3. **Важно:** Обязательно выделите каналы из списка `Included Resources`, к которым этот тариф дает доступ.
